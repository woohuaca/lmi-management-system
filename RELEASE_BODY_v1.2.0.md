# v1.2.0

`lmi-management-system` v1.2.0 开始从“生成计划文档”进一步走向“可执行的 LMI 协同系统”。

这次发版的重点，不是再多生成一些模板内容，而是补上时间设计、结构化状态和飞书日历同步链路，让周计划、日计划和真实执行之间更连贯。

## 主要更新

- 新增中文版 `LMI_SKILL_USER_GUIDE.md`
  - 帮助第一次接触这套 Skill 的用户理解它不是普通 Todo，而是一条 `角色 -> 目标 -> 计划 -> 执行 -> 复盘` 的管理闭环
  - 推荐从“先做日复盘”开始上手

- 新增 `references/intelligent-lmi-collaboration-system.md`
  - 系统化整理 LMI 协同系统的原则层、策略层、运行层
  - 明确提出“脚本先定状态，模型再做协同”的长期方向

- 周计划现在会生成结构化时间承诺
  - 新增 `scripts/lmi_time_commitments.py`
  - 周计划生成后，会把保护时间块、缓冲建议和日历同步对象写入 `memory/时间承诺/`
  - 不再只把这些信息停留在 Markdown 段落里

- 新增飞书日历同步器
  - 新增 `scripts/sync_lmi_calendar_commitments.py`
  - 支持先预览，再执行同步
  - 支持保存同步状态，避免重复创建
  - 缺少授权时，会明确提示所缺 scope 和授权命令

- 日计划与周计划加强了时间设计逻辑
  - 强调关键任务优先进入最好的连续大块时间
  - 长任务和高认知负荷任务后主动留恢复与切换缓冲
  - 固定承诺应进入飞书日历或其他稳定日历源，而不是只停留在草案

## 对使用者的实际影响

- 周计划不再只是“建议保护时间”，而是会形成可复用、可同步的结构化对象
- 日计划能更稳定承接本周已保护的大块时间，而不是每次从散乱文本里重新猜
- 飞书日历接入从“文档建议”迈进到“可执行链路”，只差一次用户授权即可真正写入
