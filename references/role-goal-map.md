# Role Goal Map

Use this reference for role clarification and role goal tables.

## Required Distinctions

- `Management role`: the value this role must create for the organization
- `Primary responsibilities`: the 3 most important things this role must carry
- `Key results`: the measurable outcomes that prove the role is working
- `High-return activities`: the few actions most likely to improve results
- `Management mechanisms`: recurring cadences, reviews, one-on-ones, dashboards, or routines that stabilize execution

## Writing Guidance

- Keep role count small. Usually 3 to 5 roles is enough.
- Responsibilities should be stated as ownership, not generic participation.
- Key results should be measurable and time-bound when possible.
- High-return activities should be specific actions, not slogans.
- Mechanisms should describe a repeatable cadence, not a one-off effort.

## Review Guidance

For each role, ask:

- Did the time investment match the stated importance?
- Were the high-return activities truly protected?
- Was the mechanism strong enough to support results?
- If results lagged, was the issue target quality, execution quality, or mechanism quality?
